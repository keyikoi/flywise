# Impeccable Design Guidelines

## Design Context

### Users
- 中国年轻旅行者（20-35岁），用手机浏览航班信息
- 核心场景：想出去玩但不确定何时买票最划算，需要价格参考和决策支持
- Job-to-be-done：快速了解航班价格水位，判断现在买还是再等等，获得省钱的信心

### Brand Personality
- **年轻** — 不是老气横秋的 OTA，是像朋友一样帮你省钱的 AI 助手
- **科技** — 数据驱动、图表可交互、价格预测有依据，让人觉得"这个 AI 真懂"
- **可信** — 价格分析有理有据，推荐透明不套路，用户看得到逻辑而不是黑盒结论

### Aesthetic Direction
- **视觉基调**: 浅色模式，干净现代，像 Google Flights 一样清爽专业但更有温度
- **参考**: Google Flights（价格趋势交互、信息层级）、ChatGPT / Arc（对话体验流畅、界面干净现代）
- **反参考**: 不要像携程/飞猪那样促销感过重；不要深色科技风（已放弃）；不要信息过载
- **色彩**: 浅色底 + 主色采用充满活力的蓝紫色系，绿色表示低价/推荐买入，红/橙色表示高价/建议等待
- **排版**: 中文优先，使用系统字体栈保证渲染质量，数字使用等宽或 Tabular 字体（如 Inter / DM Sans）
- **交互**: 价格图表必须可交互（触摸滑动看历史价格），卡片有微动效，快捷操作显眼可点

### Design Principles

1. **价格前置 (Price First)**
   每个航班卡片都同时展示价格和价格水位，用户不需要额外操作就能看到"这个价格贵不贵"。价格趋势图直接内嵌在对话流中，不藏在二级页面。

2. **一目了然 (Glanceable)**
   颜色编码代替文字说明：绿色 = 低价好时机，黄色 = 正常，红色 = 偏贵再等等。用户扫一眼就知道该不该买。

3. **可交互的数据 (Interactive Data)**
   价格图表不是装饰，是工具。用户可以触摸/滑动查看每天的历史价格，长按看详情。图表是核心功能，不是附属品。

4. **对话即决策 (Chat = Decision)**
   AI 不只是回答问题，而是主动提供决策所需的全部信息：当前价格 → 历史对比 → 趋势预测 → 明确建议。一条消息搞定。

5. **轻盈不廉价 (Light but Not Cheap)**
   浅色主题要做到干净通透但有质感。用微妙的阴影、精心的间距、克制的动效来传达品质感，而不是堆渐变和光斑。
