项目背景

航班 Tinder 应用

"我有一个 vibe coding 的想法，核心是以 tinder 那种滑动的卡片交互，卡片的内容是航班卡片，卡片上面会有起降时间 机票价格，图形化的。更进一步就是做个 tinder 版的选航班，比如你输入自己的偏好，从目的地和航班的各种纬度都给你匹配。你从专业角度帮我想想，最开始的 mvp 版本可以实现核心功能"
左右滑动卡片 偏好匹配 一键预订

核心痛点

自然语言的困境

问题

UI 调整时，自然语言不容易传达精确意图

例子 1
"把卡片间距调大一点"

→ AI 可能改错地方

例子 2
"弹窗居中"

→ 可能只水平居中

解决方案

Figma MCP 联动

方法

直接给设计截图/节点，AI 看到 exact 样式

案例 1
gap-4 → gap-6

卡片间距调整

案例 2
top-4 → top-1/2 -translate-y-1/2

弹窗垂直居中

🎯 优势：一次到位，避免多轮对话

协作技巧 2

改 Bug 的正确姿势

📸 截图大法

  • 网页报错 → 截图保存
  • 丢给 AI 自己看错误信息
  • 比文字描述高效 10 倍

AI 看图能力 > 听你描述能力

协作技巧 3

Git 版本管理

工作流

  • 每次成功改动就 commit
  • 错了随时回滚
  • commit message 作为"外部记忆"
git commit -m "feat: 添加滑动卡片动画"
git commit -m "fix: 修复卡片间距问题"
git commit -m "feat: 集成 Figma MCP"

心得总结

设计师的优势

  • 视觉敏感 — 更容易发现 AI 生成的 UI 问题
  • Figma 是共同语言 — 用设计稿跟 AI 沟通比文字更高效

AI 是强大的，但需要明确的边界和持续的提醒

Vibe Coding = 人机协作，不是机器替代
